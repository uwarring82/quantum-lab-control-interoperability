# Governance

## Stewardship model

This repository is maintained by a single steward (Ulrich Warring) as a living community document. Stewardship means:

- Maintaining the document's structure and internal consistency.
- Incorporating factual corrections promptly.
- Versioning interpretive changes explicitly.
- Preserving the document's non-prescriptive, landscape-mapping scope.

Stewardship **does not** imply:

- Institutional authority over any project described.
- Technical gatekeeping or standards-setting power.
- Unilateral authority to define interoperability standards for the ecosystem.
- Endorsement of any framework, tool, or organisation over another.

## Revision policy

| Change type | Process |
|---|---|
| Factual correction (error, outdated status, missing project) | May be incorporated directly via PR or steward edit. Noted in CHANGELOG. |
| Interpretive change (reframing, new analysis, shifted emphasis) | Versioned explicitly. Requires changelog entry with rationale. |
| Substantial addition (new section, new project coverage) | Attributed in acknowledgements or changelog. Discussed in an issue before merging. |
| Unresolved disagreement | Logged in the document or as a persistent issue — not flattened into false consensus. |

## Transition to shared governance

If the coordination activity grows beyond a single document — e.g. shared schemas, Rosetta examples, test cases, workshop outputs — a dedicated GitHub organisation with multiple maintainers may be appropriate. That transition should be decided collectively by active contributors, not unilaterally by the steward.

## Code of conduct

Contributors are expected to engage constructively, respect the neutrality of the document, and avoid using contributions as implicit advocacy for any single project. The steward reserves the right to decline changes that compromise the dossier's non-prescriptive character.
