# Contributing

Thank you for your interest in improving the Kompass-Dossier. This document is a community resource; contributions make it better.

## Types of contribution

### Factual corrections

If you find an error — wrong licence, outdated project status, incorrect attribution, broken link — please open an issue or submit a pull request. These are incorporated directly and noted in the changelog.

### Missing projects or initiatives

If a relevant open-source project or institutional initiative is not covered, please open an issue describing the project, its relationship to the ARTIQ ecosystem, and publicly accessible references. The steward will assess fit and incorporate where appropriate.

### Alternative perspectives

If you disagree with the dossier's framing, emphasis, or analysis on a specific point, please open an issue describing your perspective. The dossier prefers documented diversity of views over forced consensus. Alternative perspectives may be incorporated as noted counterpoints rather than replacements.

### New interoperability proposals

If you have a concrete interoperability idea not yet covered, please open an issue for discussion before submitting a pull request. This ensures alignment with the document's scope and avoids duplicating existing coverage.

### Rosetta examples and worked cases

Contributions of cross-framework worked examples (e.g. the same physical task expressed in multiple frameworks) are especially welcome. These should be placed in a dedicated directory and referenced from the main dossier.

## Guidelines

- Keep contributions factual and neutral. This is a landscape map, not an advocacy document.
- Cite primary sources (project documentation, papers, repositories) rather than secondary summaries.
- If in doubt about scope, open an issue before writing a full pull request.
- All contributions are licensed under CC BY 4.0, consistent with the repository licence.

## Issue templates

Use the provided issue templates for:

- **Factual correction** — errors, outdated information
- **Missing project** — a project that should be covered
- **Discussion** — interpretive questions, alternative perspectives, scope questions
